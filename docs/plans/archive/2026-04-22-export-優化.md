# 2026-04-22 匯出功能優化

## 背景

既有「檢查表匯出」存在 4 個痛點：

1. **檔名問題**：「網頁列印 → 另存為 PDF」時，瀏覽器預設檔名來自頁面 `<title>`，並非文件名稱。
2. **Excel 卡頓**：`useExport.ts` 頂部靜態 `import ExcelJS from 'exceljs'`，導致 ExcelJS（~3MB 未壓縮）整包進入首屏 main chunk，主執行緒解析時嚴重卡頓。
3. **PDF 字型缺失**：`public/fonts/NotoSansTC-Regular.ttf` 不存在，載入失敗。
4. **列印殘影**：Vuetify 3 的 `v-dialog` 會 teleport 到 `<body> > .v-overlay-container`，列印 CSS 未覆蓋，加上關閉等待不足（< 16ms），造成列印預覽仍顯示 Dialog。

## 根因

| 問題 | 根因 |
|------|------|
| 檔名 | `document.title` 固定為系統標題，瀏覽器「另存為 PDF」讀不到文件名 |
| Excel 卡頓 | 靜態 import ExcelJS 拉進 main chunk |
| PDF 字型 | 本地字型不存在，無 fallback 機制 |
| 列印殘影 | `.v-overlay-container` 不在 print CSS 隱藏清單；等待時間不足動畫完成（~225ms） |

## 解決方案

### 問題 1：檔名（`ChecklistPreviewView.vue`）

載入檢查表後設定 `document.title = \`\${data.name} - 自主檢查表\``，讓瀏覽器「另存為 PDF」對話框自動帶入文件名稱。`onUnmounted` 還原標題。

### 問題 2：Excel 卡頓（`useExport.ts`）

改用動態 import，Vite 自動將 exceljs / jspdf / jspdf-autotable 切成獨立 chunk，預覽頁首屏不再下載匯出函式庫：

```ts
async function buildExcelBlob(checklist: Checklist): Promise<Blob> {
  const { default: ExcelJS } = await import('exceljs')
  // ...
}

async function buildPdfDoc(checklist: Checklist): Promise<jsPDF> {
  const [{ jsPDF: JsPDF }, { autoTable }, fontBase64] = await Promise.all([
    import('jspdf'),
    import('jspdf-autotable'),
    loadPdfFont(),
  ])
  // ...
}
```

### 問題 3：PDF 字型（`useExport.ts`）

改從 jsDelivr CDN 動態下載 Noto Sans TC TTF（完整 TTF，jsPDF 可消化，Google Fonts CSS2 API 回傳 woff2 無法使用）：

```ts
const PDF_FONT_URL =
  'https://cdn.jsdelivr.net/npm/@expo-google-fonts/noto-sans-tc@0.4.3/400Regular/NotoSansTC_400Regular.ttf'
```

字型來源改為 `@expo-google-fonts/noto-sans-tc` npm 套件（版本鎖定 `0.4.3`），原先使用的 `notofonts/notosanstc` GitHub 倉庫路徑已不存在（404）。以 `cachedFontBase64`（module scope）快取 base64 字串，同一 session 僅下載一次。離線時顯示清楚錯誤訊息並引導改用「列印」。

### 問題 4：列印殘影（`App.vue` + `ExportDialog.vue`）

**雙重保險：**
- `App.vue` 非 scoped `@media print` 追加 `.v-overlay-container { display: none !important }`
- `ExportDialog.vue` print 分支改為 `setTimeout(resolve, 300)` 等 Vuetify Dialog 淡出動畫（~225ms）完成後再呼叫 `window.print()`

## 修改清單

| 檔案 | 修改內容 |
|------|---------|
| `src/composables/useExport.ts` | 靜態 import → 動態 import；PDF 字型改 CDN；錯誤訊息更新 |
| `src/views/ChecklistPreviewView.vue` | 載入後設定 `document.title`；`onUnmounted` 還原 |
| `src/components/ExportDialog.vue` | 列印等待改 300ms setTimeout |
| `src/App.vue` | `@media print` 追加 `.v-overlay-container` |

## 風險說明

- **CDN 可用性**：jsDelivr 為商用 CDN，穩定度高；企業內網封鎖時 Dialog 會顯示明確錯誤，使用者可改走「列印」。
- **首次 PDF 耗時**：~9MB 字型首次下載約 2–5 秒，Dialog loading 動畫給予明確反饋；瀏覽器 HTTP 快取後第二次 < 100ms。
- **Dialog 關閉延遲**：300ms 幾乎無感；若 Vuetify 未來調整 transition 時長，需同步修改此數值。
